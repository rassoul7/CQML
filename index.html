Sujet #2 : GitHub Actions (CI/CD Automatisé)

Statut actuel :

Déployé avec succès

1. Intégration Continue (CI)

Automatisation de la validation du code à chaque modification. Cela inclut l'exécution des tests unitaires et la vérification de la qualité du code pour détecter les erreurs au plus tôt.

2. Déploiement Continu (CD)

Après validation, le code est automatiquement poussé vers l'environnement de production. Cette page est un exemple concret de CD : un "git push" déclenche le workflow qui met le site en ligne.

3. GitHub Actions (L'Outil)

Moteur d'automatisation puissant de GitHub. Il utilise des fichiers YAML pour définir des "Workflows" qui réagissent aux événements de votre dépôt (comme un "push" ou une "pull request").
